# Get candidate introgressed fragments

### Input

To obtain candidate introgressed fragments from `sstar`, users should provide the output (e.g. [test.tract.threshold.tsv](https://github.com/xin-huang/sstar/blob/v1/tests/data/test.tract.threshold.tsv)) from `sstar threshold`. If no source genome is available, users can use the following command:

	sstar tract --threshold tests/data/test.tract.threshold.tsv --output-prefix test

If one source genome is available, users can calculate source match rates and output these source match rates and candidate introgressed regions in a single BED file:

	sstar tract --threshold tests/data/test.tract.threshold.tsv \
	            --output-prefix test \
				--match-rate tests/data/test.tract.src1.match.rate.tsv

If source genomes from two different source populations are available, users can provide the output (e.g. [test.tract.src1.match.rate.tsv](https://github.com/xin-huang/sstar/blob/v1/tests/data/test.tract.src1.match.rate.tsv) and [test.tract.src2.match.rate.tsv](https://github.com/xin-huang/sstar/blob/v1/tests/data/test.tract.src2.match.rate.tsv)) from `sstar matchrate`, and use the following command:

	sstar tract --threshold tests/data/test.tract.threshold.tsv \
	            --output-prefix test \
				--match-rate tests/data/test.tract.src1.match.rate.tsv tests/data/test.tract.src2.match.rate.tsv

### Output

If no source match-rate file is provided, `sstar tract` writes one BED file:

	test.bed

If one source match-rate file is provided, `sstar tract` writes one BED file containing the source match rate:

	test.bed

If two source match-rate files are provided, `sstar tract` writes two BED files:

	test.src1.bed
	test.src2.bed

These two files contain segments assigned to one of the two possible source genomes or source populations.

An example of the output is below:

<table>
	<tr>
		<td>1</td>
		<td>75030000</td>
		<td>75080000</td>
		<td>tsk_10</td>
		<td>0.280797</td>
	</tr>
</table>

- The first column is the name of the chromosome.
- The second column is the start position.
- The third column is the end position.
- The fourth column is the name of the sample.
- The fifth column is the source match rate. This column is included only when source match-rate files are provided.

### Settings

| Argument | Description |
| - | - |
| `--threshold` | Path to the threshold file generated by `sstar threshold`. |
| `--match-rate` | One or two source match-rate files generated by `sstar matchrate`. |
| `--output-prefix` | Prefix for output BED files. |
| `--diff` | Difference cutoff used to assign tracts when two source match-rate files are provided. Default: `0`. |

When two source match-rate files are provided, users can use `--diff` to set the difference cutoff for assigning tracts to the two sources. The current implementation compares `match_rate_x - match_rate_y`, where `match_rate_x` is from the first match-rate file and `match_rate_y` is from the second match-rate file. Rows with `match_rate_x - match_rate_y > diff` are written to `.src1.bed`, and rows with `match_rate_x - match_rate_y < diff` are written to `.src2.bed`.
